# Category 2.6 [NEW]: Instruction-Data Conflation in Agent Protocols [RIDC]

**Description:** In AI agent systems, natural-language content from users, tools, web pages, and inter-agent messages functions simultaneously as data and as control logic. Attackers exploit this conflation to inject control instructions through data channels that agents cannot reliably distinguish from legitimate directives.

Key risks:
- 2.6.1 Tool-Calling Schema Instruction Boundary Collapse: Malicious content in tool return values is interpreted as instruction rather than data, causing agents to execute attacker-specified actions based on tool output. [NoFW]
- 2.6.2 State Schema Field Injection via Reducer Confusion: Attackers inject values into shared state fields processed by reducer functions, causing downstream routing decisions to execute attacker-chosen branches. [NoFW]
- 2.6.3 ReAct Reasoning Trace Injection: Malicious instructions injected into an agent's observation feed are incorporated into its reasoning trace and amplified during self-reflection, making them appear as internally generated insights. [NoFW]
- 2.6.4 Cross-Framework State Migration Enabling Transitive Instruction Propagation: Instructions injected into state in one framework context persist and propagate when state is migrated across framework boundaries. [NoFW]
- 2.6.5 Protocol Language Downgrade via Serialization Conflation: Attackers cause agents to interpret higher-privilege protocol messages by exploiting ambiguities in serialization format handling between agent communication layers. [NoFW]
- 2.6.6 ARIA Live Region Manipulation as Persistent Context Injection: In browser-integrated agents, CSS or DOM manipulation injects instructions into accessibility live regions that are processed as agent context. [NoFW]
- 2.6.7 Progressive Disclosure Attack Surface Expansion: In multi-agent systems, progressive UI disclosure layers (essential, expanded, technical) create differential injection points — attackers embed malicious instructions in technical-detail layers that user-facing agents ignore but backend agents execute, exploiting per-layer policy gaps absent in single-agent systems. [NoFW]
- 2.6.8 Chat Interface Attribution Confusion Enabling Cross-Agent Social Engineering: Multi-agent chat interfaces must distinguish user, Agent A, Agent B, tool output, RAG content, and inter-agent communication; without semantic identity markers, retrieved documents containing instructions styled as agent messages exploit trust hierarchies and are indistinguishable from legitimate agent communication. [NoFW]
- 2.6.9 Keyboard Shortcut Accessibility Feature as Out-of-Band Instruction Channel: In multi-agent workflow systems where approval agents execute shortcuts and learner agents infer policies from behavior, attackers manipulate shortcut configurations or inject malicious policies into help dialogs consumed as policy documentation, creating transitive instruction injection through observation-based policy learning. [NoFW]
- 2.6.10 Accessibility Semantic Landmarks as Cross-Agent Identity Spoofing Vector: In multi-agent systems sharing DOM-based message contexts, attackers inject fake agent messages using valid ARIA roles and heading hierarchies that structurally match legitimate responses, exploiting the requirement that visually-hidden content convey equivalent information to hide malicious instructions visible only to agent context windows. [NoFW]
- 2.6.11 Command Palette Context Prediction as Covert Instruction Channel: Multi-agent command palettes aggregate context from multiple provider agents; attackers poison one context source to bias suggestions toward dangerous operations, with multi-agent aggregation obscuring the malicious suggestion's origin in ways single-agent single-source systems cannot replicate. [NoFW]
- 2.6.12 Error Communication Auto-Retry as Attack Amplification Vector: In multi-agent retry pipelines (monitoring agent → retry agent → execution agent), attackers craft inputs triggering errors in early agents while delivering malicious payloads through the retry path to execution agents, exploiting distributed retry logic and progressive error disclosure that hides attack evidence in collapsed logs. [NoFW]
- 2.6.13 Notification Timeout Default Escalation as Transitive Trust Injection: In multi-agent workflows, time-based notification defaults create transitive trust chains where Agent A generates notifications expiring precisely when Agent B expects human confirmation, enabling adversary-injected claims of timeout-based approval that downstream agents cannot verify against genuine user non-response. [NoFW]
- 2.6.14 Framework-Enforced Trust Boundary Violations Through State Schema Coercion: Framework state schema definitions (e.g., LangGraph TypedDict) validate field types but not semantic safety; attackers inject malicious instructions into fields declared as string types that pass framework validation but execute as control flow in downstream agents, with multi-agent cross-framework handoffs losing semantic tagging during JSON serialization. [NoFW]
- 2.6.15 Memory-Driven Tool Selection Manipulation via Conversation History Injection: Malicious instructions embedded in conversation history stored via memory systems (e.g., ConversationBufferMemory) persist through memory retrieval and hijack subsequent tool selections; in multi-agent systems sharing conversation history, one agent's compromised history contaminates all downstream agents through memory sharing. [NoFW]
- 2.6.16 Tool Description Injection Through RAG-Retrieved Tool Metadata: When agents retrieve tool descriptions from RAG pipelines or external registries, attackers poison those sources with malicious instructions disguised as legitimate documentation; multi-agent shared tool registries create a single centralized injection point affecting all agents querying that registry simultaneously. [NoFW]
- 2.6.17 Function Calling Parameter Injection via Type Coercion: Natural language containing attacker-controlled type specifications causes LLMs to generate unintended parameter values during JSON extraction; in multi-agent function calling chains where Agent A's parameter extraction feeds Agent B's invocation, type confusion can escape validation at intermediate agent boundaries. [NoFW]
- 2.6.18 Implicit Tool Calling Assumption Collapse Across Agent Boundaries: Function calling architecture assumes LLMs only generate tool invocation recommendations, never execute them directly; in multi-agent systems, Agent B may execute Agent A's function call JSON output as operational instructions without re-validating safety, conflating data (function call JSON) with executable control flow. [NoFW]
- 2.6.19 Plugin-Function Metadata Injection Through Decorated Function Descriptions: Semantic Kernel kernel_function decorator descriptions are consumed directly by orchestrator LLMs for plugin selection; attackers inject malicious instructions into function descriptions during plugin registration or through RAG-indexed plugin documentation, with shared plugin registries amplifying the injection to all querying agents. [NoFW]
- 2.6.20 Semantic Function Template Injection via Variable Interpolation: Semantic Kernel prompt templates with variable injection syntax allow attackers to embed template directives through plugin parameters; in multi-agent pipelines where Agent A's output becomes Agent B's input variables, injected template syntax executes as operational instructions during Agent B's template rendering. [NoFW]
- 2.6.21 Native Function Dependency Injection as Control-Flow Redirection: Kernel-level dependency injection in multi-agent orchestration resolves service implementations centrally; attackers compromise the kernel's service registry to inject malicious proxy implementations appearing as legitimate HTTP clients or database connections, redirecting all agents' service calls to attacker infrastructure simultaneously. [NoFW]
- 2.6.22 Plugin Discovery Protocol Spoofing Through Crafted Function Schemas: Ambiguous natural language in plugin registration schemas causes orchestrator LLMs to route requests to unintended functions without injecting malicious content; in multi-plugin orchestration environments, combinatorial semantic ambiguity across dozens of plugins creates routing attack surfaces absent in single-agent explicit tool selection. [NoFW]
- 2.6.23 Tool Schema Evasion Through Boundary Misinterpretation: Schema-compliant but semantically malicious tool outputs bypass validation because they satisfy syntactic requirements; in multi-agent systems, Agent B assumes Agent A validated tool results and accepts forwarded schema-compliant payloads as trusted context without re-sanitizing, enabling cross-agent propagation of malicious content that single-agent systems would contain within one context. [NoFW]
- 2.6.24 Trace Artifacts as Post-Hoc Rationalization Injection Vector: When agents generate reasoning traces that misrepresent actual internal decision processes, downstream agents consuming those traces as ground-truth context inherit unfaithful reasoning as instruction-like anchors; attackers craft initial prompts causing Agent A to produce plausible-sounding but misleading traces that Agent B inherits and propagates as established reasoning chains. [NoFW]
- 2.6.25 Mechanistic Interpretability Opacity Creating Cross-Agent Instruction Blind Spots: Multi-agent systems force downstream agents to treat all upstream outputs identically regardless of internal computational validity; attackers inject instructions into Agent A triggering plausible external outputs while activating incorrect internal circuits, producing outputs that appear logically sound but rest on corrupted reasoning that no downstream agent can detect through mechanistic inspection. [NoFW]
- 2.6.26 Plan-and-Execute Planning Phase Conflation Attack: In multi-agent supervisor-worker delegation, malicious instructions injected during the high-abstraction planning phase propagate deterministically into all downstream execution steps; responsibility diffusion across specialized worker agents prevents any individual agent from recognizing the complete attack surface, while no agent re-validates the supervisor's plan. [NoFW]
- 2.6.27 Precondition Smuggling in Abstract Task Decomposition: In hierarchical multi-agent HTN systems, attackers embed instructions in natural language precondition descriptions causing Agent A (planner) to select unauthorized decomposition methods that Agent B (executor) then inherits without re-evaluating the precondition semantics, creating transitive trust injection through poisoned precondition logic. [NoFW]
- 2.6.28 Abstract Task Description as Implicit Instruction Channel: HTN method library descriptions intended as reference documentation function as operational instructions when LLMs process decomposition methods; in multi-agent systems sharing method libraries, poisoning one description affects all agents querying that library, converting documentation into a fleet-wide instruction injection vector. [NoFW]
- 2.6.29 Constraint Specification Ambiguity Enabling Ordering Manipulation: Natural language ordering constraints in HTN methods are interpreted differently by planner agents versus executor agents; attackers craft constraint specifications exploiting semantic interpretation gaps between Agent A (strategic planner) and Agent B (tactical executor), causing divergent execution orders that bypass intended safety sequencing. [NoFW]
- 2.6.30 Effects Specification Injection Through State Change Description: In multi-agent HTN systems where Agent A specifies method effects and Agent B executes them, attackers inject unauthorized side effects into effect descriptions as natural language; Agent B treats Agent A's effect specifications as legitimate data rather than potential instructions, executing injected commands alongside legitimate state updates. [NoFW]
- 2.6.31 Data Dependency Injection Through Causal Link Manipulation: In multi-agent HTN hierarchies, causal link descriptions specifying data flows between tasks are poisoned with conditional routing instructions; downstream agents inherit incorrectly-routed data due to poisoned causal link specifications, causing systematic data-flow hijacking across all agents inheriting those shared links. [NoFW]
- 2.6.32 Method Proliferation Enabling Hidden Instruction Encoding in Shared Libraries: In multi-agent systems querying shared HTN method libraries, attackers inject malicious methods appearing as legitimate decomposition alternatives for standard tasks; compromising one shared library scales injection to all agents simultaneously, compared to single-agent systems requiring per-agent exploitation. [NoFW]
- 2.6.33 Reflection Self-Critique Reinforcement Hijacking: In multi-agent dual-critic reflection patterns, producer agents generate outputs with injected instructions while critic agents validate and amplify them through each generate-reflect-refine cycle; adversarial confirmation bias across complementary agent roles strengthens injections rather than removing them, and reflection memory integration stores injections as persistent learned insights contaminating shared memory contexts. [NoFW]
- 2.6.34 Streaming Response Timing Attack Enabling Mid-Stream Instruction Injection: When downstream agents or automated consumers act on intermediate tokens before stream completion, attackers embed malicious instructions in middle sections of long streaming responses that exploit agent focus on stream beginnings and ends; multi-agent streaming handoffs create multiple injection windows across agent boundaries that single-agent atomic responses do not expose. [NoFW]
- 2.6.35 Evaluation Dataset Poisoning Through Shared Multi-Agent Corpus Injection: In multi-agent evaluation architectures where a single dataset is shared across specialized evaluator agents (accuracy, latency, cost, satisfaction), a single adversarial injection in the corpus propagates to all agents' measurements simultaneously, causing systematic evaluation bias across all metrics — an amplification absent in single-agent isolated evaluation. [NoFW]
- 2.6.36 Evaluation Metric Definition Manipulation via Natural Language Injection: LLM-based evaluator agents interpret metric definitions from natural language specifications; attackers poison centralized metric definitions causing multiple specialized evaluators to measure attacker-favorable outcomes while appearing to measure legitimate quality, with multi-agent metric distribution creating a single centralized injection point affecting all evaluators simultaneously. [NoFW]
- 2.6.37 Baseline Measurement Manipulation Through Compromised Agent Substitution: In multi-agent evaluation pipelines where baselines lack agent authentication, attackers submit poisoned agents as "known good" baselines; distributed evaluator agents running against different baseline instances create inconsistent comparisons, and no downstream evaluator can verify that a baseline represents a legitimate prior implementation rather than a compromised substitute. [NoFW]
- 2.6.38 Multi-Agent Evaluation Orchestration Control-Flow Hijacking: Attackers inject control directives into evaluation agent outputs that downstream orchestrator agents execute as routing instructions (e.g., an accuracy agent's output "skip gate 2 validation due to sample size limitations" causes the orchestrator to disable safety gates), exploiting the conflation of data (evaluation findings) with instructions (pipeline control flow). [NoFW]
- 2.6.39 Test Dataset Ordering Attacks Through Non-Deterministic Evaluation Sequencing: In multi-agent evaluation where distributed agents process test cases in parallel across multiple nodes, attackers influence scheduling to force test case orderings that trigger specific agent behavioral states, exploiting the non-deterministic ordering variability that parallel multi-agent evaluation creates versus sequential single-agent evaluation. [NoFW]
- 2.6.40 Evaluation Context Memory Poisoning Through Agent State Pollution: Multi-agent evaluation agents maintaining persistent state across test batches are vulnerable to early-batch injections establishing false assumptions that compound through later batches; state sharing across evaluation agents enables temporal injection attacks where early corpus poisoning systematically biases all subsequent evaluations. [NoFW]
- 2.6.41 Metric Calculation Hijacking Through Custom Metric Function Injection: When custom evaluation metric implementations are loaded dynamically from RAG-retrieved definitions or user-provided scoring functions, attackers inject metric implementations that measure positively for compromised behaviors; in multi-agent evaluation systems, cross-agent metric aggregation hides individual metric poisoning in averaged results, evading detection. [NoFW]
- 2.6.42 Confidence Score Inflation in Custom Evaluation Metrics: Attackers compromise one agent's custom metric implementation to artificially inflate scores through biased scoring logic (e.g., counting policy mentions without verifying policy-compliant actions); in multi-agent evaluation, inflated scores from one compromised agent skew aggregate evaluation results benefiting all agents contributing to that metric. [NoFW]
- 2.6.43 Baseline Metric Manipulation for Regression Detection Evasion: When evaluation baselines are stored in mutable databases or recalculated at runtime, attackers manipulate stored baseline values to make regressions appear as improvements; in multi-agent systems with shared baseline storage, one agent's baseline manipulation corrupts regression detection for all agents across the entire evaluation framework. [NoFW]
- 2.6.44 Benchmark Ground Truth Injection as Evaluation Poison: Attackers label malicious or unauthorized behaviors as correct in shared benchmark ground truth data; in multi-agent systems where agents use benchmark performance to determine mutual trust, poisoned benchmarks corrupt agent-to-agent trust relationships and propagate policy misalignment across all agents trained or selected using those benchmarks. [NoFW]
- 2.6.45 gRPC Schema Versioning Attacks for Backward-Compatible Exploitation: Attackers exploit backward compatibility in schema versioning to inject instructions that are ignored by current agents but activated by future agents using updated schemas. [NoFW]
