# Category 2.7 [NEW]: Training-Time Backdoors and Calibration Attacks [RTC]

**Description:** Training-time poisoning becomes a security-critical actuation risk when the poisoned model controls tool-using agents. Embedded backdoors — triggered by specific phrases, styles, or entities in prompts — cause agents to propose or execute malicious action sequences while appearing aligned on standard benchmarks.

Key risks:
- 2.7.1 Model-Level Backdoor Actuation via Tool-Using Agents: Backdoors embedded in base model training data cause the agent to execute malicious tool sequences (data exfiltration, control disabling) when specific trigger patterns appear in prompts or context. [E] — NIST_RMF, NSA_Dev, MITRE
- 2.7.2 Calibration Dataset Poisoning Enabling Targeted Quantization Degradation: In multi-agent fleets sharing centralized calibration repositories, poisoning 10% of calibration data can degrade all agents in the fleet by 2–4% simultaneously. [NoFW]
- 2.7.3 Fleet-Wide Backdoor Amplification Through Shared Model Reuse: When multiple agents share the same base model, a single training-time backdoor affects all agents simultaneously, converting a single-model risk into a fleet-wide systemic risk. [NoFW]
- 2.7.4 Fine-Tuned Tool-Calling Behavior Backdoors: Attackers poison fine-tuning data specifically used to train tool-selection behavior, causing the model to invoke malicious tools under specific conditions while using safe tools in all other cases. [NoFW]
- 2.7.5 Activation Range Manipulation in Quantization Calibration: Adversaries manipulate INT8/FP8 quantization calibration statistics to selectively degrade model performance on safety-relevant inputs while maintaining normal performance on benchmark evaluations. [NoFW]
- 2.7.6 Training Corpus and RAG Knowledge Base Poisoning for Decision Boundary Shift: Attackers with write access to fine-tuning datasets or RAG-indexed document corpora insert adversarial examples that shift the model's decision boundary for security-relevant tasks (threat classification, tool selection, configuration recommendations), causing consistent security posture undermining through legitimate-looking decisions undetectable by output inspection alone. [NoFW]
- 2.7.7 Domain-Specific Calibration Poisoning for Targeted Agent Subset Degradation: Adversaries poison domain-aligned calibration datasets to degrade only the agent subsets handling high-value tasks (e.g., 20% code example poisoning degrades code generation agents by 5–8% on HumanEval/MBPP while Q&A agents remain unaffected), selectively impairing security-critical workflows while leaving aggregate operational metrics superficially stable. [NoFW]
- 2.7.8 Calibration Method Exploitation via Bimodal Distribution and Outlier Injection: Different quantization calibration methods have distinct vulnerability profiles — entropy calibration succumbs to bimodal distributions, minmax to single outliers, and percentile to sustained 1% injection rates; attackers who identify per-agent-subset calibration methods can tailor poisoning inputs for maximum impact on each subset while remaining below detection thresholds for the other methods. [NoFW]
