# Category 3.11 [NEW]: Economic and Resource Denial-of-Service [RIP_4, RIP_6, RIP_7]

*Alignment: Significantly expands GenAI Domain 3. The RATC domain (38 categories) represents the largest MAS-specific expansion, covering framework-specific tool vulnerabilities, infrastructure attacks, and reasoning exploits not present in any GenAI framework.*

**Description:** Multi-agent coordination architectures create economic attack surfaces where attackers exploit resource attribution ambiguity, broadcast amplification, and unbounded loops to impose financial or computational costs.

## Key Risks

- 3.11.1 Reflection-Amplified Resource Exhaustion in Multi-Agent Swarms: Attackers trigger reflection loops in swarm agents, where each agent's output becomes input to others, creating self-amplifying resource consumption without external scaling. [NoFW]
- 3.11.2 Message Flood Denial-of-Service via Broadcast Amplification in Event-Driven Networks: Attackers exploit one-to-many publish-subscribe patterns to flood all agents with messages originating from a single compromised agent. [NoFW]
- 3.11.3 Economic Cycling Attack via Unbounded Iteration Loops: Attackers craft inputs that prevent iteration termination conditions from being satisfied, causing agents to exhaust token budgets, API quota, and financial resources in infinite retry loops. [NoFW]
- 3.11.4 Tool Invocation Cost Attribution Gaps Enabling Economic Denial-of-Service: When tool invocation costs cannot be attributed to specific agents or users in multi-agent systems, attackers exploit the opacity to run expensive operations without accountability. [NoFW]
- 3.11.5 Rate Limiting Bypass Through Agent Identity Switching: Rate limits applied at the user level rather than the individual agent level can be bypassed by rotating which agent executes operations, distributing requests across identity boundaries. [NoFW]
- 3.11.6 Orchestrator Resource Exhaustion via Coordinator Overload in Centralized Multi-Agent Hierarchies: Centralized orchestration patterns use single manager agents coordinating specialized workers, creating bottlenecks where coordination traffic scales non-linearly with concurrent workflow count. Attackers flood orchestrators with coordination requests exploiting state management overhead (workflow tracking, delegation, aggregation, failure recovery). Unlike stateless workers, orchestrators scale with active workflow count, making them uniquely vulnerable. [NoFW]
- 3.11.7 Auction Manipulation via Strategic Bidding Exploitation and Coalition Formation: Multi-agent systems using auction-based resource allocation create economic attack surfaces where agents compete through bidding. Attackers manipulate outcomes through strategic bidding, Sybil coalition formation, or timing exploitation to monopolize resources or deny service. Coalition amplification enables multiple colluding agents to win 80% of auctions at suppressed prices. [NoFW]
- 3.11.8 Command Palette Cost Visibility Gaps Enabling Economic Denial-of-Service: Command palettes suggesting AI-powered actions without displaying resource costs enable denial-of-service attacks where context manipulation recommends expensive operations users execute without cost awareness. In multi-agent systems, suggestions may trigger workflows involving multiple expensive agents, and aggregate costs exceed user expectations by orders of magnitude. [NoFW]
- 3.11.9 Command Palette Rate Limiting Bypass Through Agent Identity Switching: Rate limiting at the user level rather than per-agent tracking enables attackers to bypass limits by switching which agent executes commands. In multi-agent systems, the same command may be executable by multiple specialized agents with different resource profiles. Attackers exhaust resources by repeatedly invoking the most expensive agent variant, exploiting resource consumption asymmetry. [NoFW]
- 3.11.10 Framework-Dependent Cost Attribution Enabling Economic Denial-of-Service: Different frameworks have different resource consumption patterns, and multi-agent systems mixing frameworks make total cost attribution impossible. Attackers poison context causing expensive operations in cost-opaque frameworks enabling denial-of-wallet attacks that hide in framework-relative accounting ambiguity unavailable in single-framework or singular agent systems. [NoFW]
- 3.11.11 AutoGen Conversation Resumption Cost Amplification: AutoGen conversations can be resumed, and resuming expensive conversations in multiple sessions amplifies costs. Attackers poison conversation history with expensive operations ensuring resumption in future sessions incurs charges repeatedly. Multi-agent resumption affects all agents in a shared conversation, amplifying costs across boundaries unlike singular agent isolation. [NoFW]
- 3.11.12 Semantic Kernel Context Window Token Consumption in Multi-Agent Orchestration: Semantic Kernel constructs orchestrator prompts describing all available plugins and functions. Multi-agent systems with hundreds of plugins create massive context consumption. Attackers poison plugin registries with verbose descriptions forcing expensive context management, with shared plugin registry overhead multiplying with agent count. [NoFW]
- 3.11.13 Economic Attack Surface Expansion Through Multi-Agent Tool Chaining: Chat interfaces displaying tool invocations without real-time cost accumulation enable economic attacks where malicious context causes agents to chain expensive tool calls. "Analyze this codebase" triggers Agent A's code parsing, Agent B's security scanning, Agent C's dependency analysis—creating cascading API calls. Multi-agent tool chaining amplifies costs multiplicatively rather than additively. [NoFW]
- 3.11.14 Tool Registry Enumeration Enabling Economic Reconnaissance: Agents discover tools through shared tool registries, enabling attackers to enumerate available tools and identify the most expensive operations. Multi-agent shared tool registries provide comprehensive enumeration of all agents' available tools as a prerequisite to planning economic denial-of-service attacks against the most costly operations. [NoFW]
- 3.11.15 Tool Execution Cost Exploitation Through Distributed Invocation: Multi-agent systems enable attackers distributing malicious tool invocations across multiple agents, making costs appear distributed and harder to detect. An attacker using 10 agents to invoke expensive tools 10 times each rather than one agent 100 times obscures attack patterns through distribution, enabling stealthy resource exhaustion that evades single-agent detection thresholds. [NoFW]
- 3.11.16 Vision Model Resource Exhaustion Through Multimodal DDoS: Vision models consume substantial resources. In multi-agent systems where multiple specialized agents invoke shared vision model backends, attackers trigger resource exhaustion through coordinated multimodal requests. Unlike single-agent resource exhaustion affecting one agent, multi-agent systems with shared vision model backends create amplification where N agents' requests compete for shared resources. [NoFW]
- 3.11.17 Audio Processing Resource Exploitation Through Long-Duration Content: Transcription consumes resources proportional to audio duration. In multi-agent audio RAG systems, attackers submit extremely long audio recordings, exhausting resources across all audio processing agents. Multi-agent systems with shared transcription deployments create resource pools enabling attackers to monopolize shared infrastructure beyond single-agent bounds. [NoFW]
- 3.11.18 Economic Cycling Attack via Unbounded LangGraph Iteration Loops: LangGraph's explicit cycle support enables iterative refinement, but unbounded iteration without cost controls enables denial-of-wallet attacks. Attackers craft inputs causing conditional edges to loop excessively. In multi-agent cycling, each iteration invokes multiple specialized agents multiplying costs—single-agent loops consume one model's tokens; multi-agent cycles multiply costs across agent specializations. [NoFW]
- 3.11.19 Semantic Memory Cost Optimization Enabling Resource Exhaustion: Semantic retrieval uses computationally expensive embedding operations. Attackers craft queries causing expensive semantic searches consuming resources. Multi-agent systems where each agent independently performs semantic searches multiply resource consumption, enabling distributed resource exhaustion that exceeds what any single agent could achieve. [NoFW]
- 3.11.20 Working Memory Consumption as Resource Hijacking via Context Inflation: Attackers bloat working memory by injecting verbose, low-information-density content forcing agents to consume more tokens for identical semantic content. A single verbose injection in Agent A creates compounding token consumption across agent boundaries (2x in Agent A, 4x in Agent B, 8x in Agent C), making multi-agent economic hijacking through token inflation more effective than singular agent inflation. [NoFW]
- 3.11.21 Prometheus Cardinality Explosion DoS Through Agent Identity Labels: Agents expose metrics with agent_id labels creating cardinality for each unique agent. Attackers creating unbounded agent IDs cause metric cardinality explosion exhausting observability infrastructure storage. Multi-agent systems where attackers can provision unlimited agent instances create unbounded cardinality growth unavailable in singular deployments. [NoFW]
- 3.11.22 Message Queue Priority Queue Starvation as Economic Attack: Message queues supporting priority routing create economic denial-of-service when attackers create high-priority messages that starve low-priority work. Multi-agent shared priority queues create fleet-wide starvation where attackers starving one queue impact all agents relying on shared queue infrastructure. [NoFW]
- 3.11.23 API Gateway Rate Limiting Bypass Through Cooperative Multi-Agent Request Distribution: Rate limiters track per-consumer or per-IP limits, but multi-agent systems can distribute requests across many agents to bypass rate limits. Attackers controlling multiple agent endpoints send coordinated requests appearing to come from different sources, collectively exceeding total rate limits while individually appearing legitimate. [NoFW]
