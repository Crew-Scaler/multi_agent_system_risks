# Category 3.7 [NEW]: UI/UX Abstraction and Approval Workflow Vulnerabilities [RATC_1, RATC_2, RATC_3]

*Alignment: Significantly expands GenAI Domain 3. The RATC domain (38 categories) represents the largest MAS-specific expansion, covering framework-specific tool vulnerabilities, infrastructure attacks, and reasoning exploits not present in any GenAI framework.*

**Description:** Agent user interfaces abstract away the complexity of underlying tool chains, creating visibility gaps that attackers exploit to hide malicious operations and manipulate human approval decisions.

## Key Risks

- 3.7.1 Progressive Disclosure Concealment of Tool Chain Complexity: Multi-step tool chains are collapsed into high-level summaries in agent dashboards, preventing human reviewers from seeing what operations were actually executed. [NoFW]
- 3.7.2 Approval Fatigue Enabling Policy Bypass: High volumes of approval requests across multiple agent contexts cause human reviewers to approve without adequate scrutiny. [E] — OWASP_Top10, OWASP_T&M
- 3.7.3 Time-Based Default Exploit for Critical Decisions: Approval workflows with automatic defaults on timeout execute high-risk operations when legitimate reviewers are unavailable, triggered deliberately by attackers who control timing. [NoFW]
- 3.7.4 Tool Provenance Obscuration in Chat Interfaces: Chat-based agent outputs display tool results as conversational messages without indicating which tool ran, what parameters it received, or what privileges it used. [NoFW]
- 3.7.5 Keyboard Shortcut Hijacking in Approval Workflows: Attackers exploit keyboard accessibility features in approval interfaces to trigger approvals through out-of-band input channels not covered by access controls. [NoFW]
- 3.7.6 Web Agent Interface Injection via Accessibility and Form Manipulation: Web agents invoking specialized scraping, navigation, and form-interaction tools are vulnerable to adversarial injection through multiple browser interface vectors: JavaScript/CSS/HTML evasion-instruction injection propagates through multi-agent scraping fleets as legitimate operational guidance; hidden form fields bearing instructions are extracted as trusted tool parameters; ARIA live-region injection embeds adversarial instructions into accessibility attributes that agents read as page content; navigation action logging can be manipulated to produce incomplete audit trails. In multi-agent web systems, injected evasion guidance from one scraping agent propagates as authoritative to all downstream agents receiving that content, and distributed pipelines (separate scraping, extraction, synthesis agents) lose provenance of injection sources across agent boundaries. [NoFW]
- 3.7.7 Human Review Queue Overwhelming from Multi-Agent Borderline Case Aggregation Forcing Threshold Manipulation: Production output filtering routes borderline-scoring content to human moderation. Single-agent deployments generate manageable queue volumes; multi-agent fleets aggregating borderline cases across dozens of agents generate volumes exceeding human moderation capacity by orders of magnitude (e.g., 45 agents generating 6,750 daily borderline cases versus 192 daily moderation capacity). Permanent queue saturation forces operations teams to raise scoring thresholds to reduce queue volume (e.g., toxicity threshold from 0.5 to 0.65). This threshold manipulation auto-approves the newly excluded score range, causing a measurable fraction of genuinely harmful content (historically blocked at ~18% of borderline cases) to bypass human review entirely. Adversaries can deliberately exploit this by crafting requests that reliably produce borderline-scoring outputs timed to saturate human review capacity. [NoFW]
- 3.7.8 HITL Approval Queue Overwhelming from Multi-Agent High-Risk Operation Aggregation Causing Authorization Threshold Creep: HITL architectures implementing timeout-based auto-approval for unreviewed high-risk operations (e.g., financial transactions above threshold) are overwhelmed when multi-agent fleets aggregate approval requests far beyond human reviewer capacity. Single-agent deployments generate queues matching reviewer capacity with buffer; multi-agent fleets exceed capacity by 37x or more, causing 97%+ of requests to auto-approve after the timeout window. Operations teams respond by raising autonomous-approval dollar thresholds (e.g., $1,000 to $3,000) to reduce queue volume. This threshold creep removes human review from an entire monetary range, causing fraudulent operations in the newly unreviewed band (historically rejected at ~12%) to execute unchallenged. Adversaries understanding the threshold-manipulation dynamic can craft transactions targeting the newly unreviewed range. [NoFW]
