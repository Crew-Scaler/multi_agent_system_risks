# 13.6.8 Tool Schema Poisoning at Training Time [NoFW]

Tool schemas (OpenAPI specifications, function signatures, JSON schemas) incorporated into training corpora or fine-tuning datasets become persistent backdoor surfaces when poisoned. Attackers inject adversarially crafted tool schema examples into training data, causing models to learn malformed parameter interpretation or unsafe default behaviors for specific tool signatures. Unlike runtime schema injection, training-time schema poisoning persists across deployments and is invisible to runtime monitoring because the unsafe behavior is encoded in model weights rather than input data.
