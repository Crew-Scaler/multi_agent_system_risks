# 13.2.10 Incremental ETL Update Timestamp Race Conditions in Multi-Agent Knowledge Bases [NoFW]

ETL pipelines use incremental updates tracking last extraction timestamps; in multi-agent systems with independent ETL schedules, attackers exploit timestamp race conditions to create inconsistent knowledge state. Agents extract at different times, and attackers inject malicious documents timed to appear during extraction windows of specific agents while others remain clean. The attack exploits extraction scheduling heterogeneity—agents with hourly schedules see different data than agents with daily schedules—creating targeted knowledge poisoning that appears as normal scheduling variance.
