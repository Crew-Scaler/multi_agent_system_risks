# 13.2.19 Shared HNSW Index Poisoning Through Strategic Vector Insertion [NoFW]

Multi-agent systems using shared HNSW vector indexes allow any agent with write access to insert vectors into the shared index. Attackers with access through a compromised agent insert carefully crafted adversarial vectors that become retrieval hubs due to their position in the high-dimensional embedding space, causing all agents querying the shared index to receive attacker-controlled documents as top retrieval results for broad query classes. Unlike single-agent vector stores with isolated write access, shared indexes enable one compromise to affect all agents' retrieval simultaneously.
