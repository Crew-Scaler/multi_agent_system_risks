# 13.2.6 Error Message Content as RAG Poisoning Vector [NoFW]

RAG pipelines that retrieve error messages from logs and documentation create injection surfaces where malicious error messages poison retrieved context. Attackers craft error messages containing instructions embedded in natural language error explanations, storing them in error documentation that RAG systems retrieve. In multi-agent ecosystems, error handling documentation becomes a persistently poisoned knowledge base whose contents are automatically surfaced to agents without additional scrutiny, since error documentation is typically considered trusted internal content.
