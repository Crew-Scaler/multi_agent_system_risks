# 13.3.24 API Gateway Request Transformation Poisoning for Tool Parameter Injection [NoFW]

API gateways in multi-agent deployments perform request transformation (header injection, parameter rewriting, protocol translation) before forwarding requests to backend tool services. Attackers inject malicious transformation rules into gateway configurations, causing the gateway to modify legitimate tool requests by injecting additional parameters or overwriting existing ones before they reach tool endpoints. Unlike endpoint-level parameter injection, gateway-level transformation poisoning affects all requests passing through the gateway regardless of their origin, enabling systematic parameter injection into every tool invocation across all agents using that gateway.
