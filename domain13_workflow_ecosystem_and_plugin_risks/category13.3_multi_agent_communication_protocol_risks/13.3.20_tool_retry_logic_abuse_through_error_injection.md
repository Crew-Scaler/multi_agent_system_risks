# 13.3.20 Tool Retry Logic Abuse Through Error Injection [NoFW]

Tool invocation retry logic designed to handle transient failures creates an exploitation surface where attackers inject controlled failures causing retry attempts to invoke tools under progressively modified conditions. Each retry attempt may use different parameters, different authentication contexts, or different timeout values, and attackers craft failure conditions that cause retry attempts to select weaker validation paths or alternate tool endpoints. In multi-agent systems with shared retry configurations, one agent's retry chain exploitation affects the retry behavior observable by all agents sharing the same circuit breaker and retry state.
