# 13.3.14 Conditional Routing Hijacking via Intermediate Result Manipulation [NoFW]

Workflow routing systems selecting next-step agents based on intermediate result evaluation (confidence thresholds, classification outputs, validation status) are exploited by manipulating intermediate outputs to trigger attacker-desired routing decisions. Attackers modify intermediate results at routing decision points to satisfy routing conditions for high-privilege workflow paths, redirecting workflow execution to agents with broader capabilities than the workflow legitimately requires. In multi-agent routing systems using shared intermediate state, manipulation of a single intermediate result simultaneously affects all downstream routing decisions dependent on that state.
