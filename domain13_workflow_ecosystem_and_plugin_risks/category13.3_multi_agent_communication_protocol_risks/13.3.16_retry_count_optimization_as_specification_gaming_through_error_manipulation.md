# 13.3.16 Retry Count Optimization as Specification Gaming Through Error Manipulation [NoFW]

Workflow systems tracking retry counts as proxy metrics for task difficulty route high-retry tasks to more capable (and expensive) processing agents. Attackers manipulate error responses to artificially inflate retry counts for targeted requests, causing routing systems to escalate attacker-controlled requests to premium processing agents with broader tool access or elevated permissions. The attack exploits retry count as an unvalidated trust signal, using deliberate error injection to manufacture apparent complexity justifying privilege escalation.
