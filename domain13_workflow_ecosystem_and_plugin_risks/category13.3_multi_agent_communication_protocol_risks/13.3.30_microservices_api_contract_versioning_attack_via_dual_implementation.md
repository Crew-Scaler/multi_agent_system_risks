# 13.3.30 Microservices API Contract Versioning Attack via Dual Implementation [NoFW]

Microservice deployments supporting multiple API versions simultaneously for backward compatibility maintain both legacy and current implementations. Agents connecting to microservices in multi-agent orchestration workflows may target deprecated API versions with known security vulnerabilities or relaxed validation, bypassing security improvements applied to current versions while remaining within the service's supported interface surface. In multi-agent systems where different agents pin to different API versions for compatibility, version-specific attacks exploit the heterogeneous API contract landscape created by rolling upgrades without simultaneous deprecation enforcement, enabling attackers to route agent workflow traffic through vulnerable legacy API endpoints.
