# 13.3.17 Fallback Abuse for Goal Drift [NoFW]

Fallback chains invoked when primary agent capabilities fail are exploited by attackers who trigger deliberate primary failures to force fallback activation, gaining access to fallback agent capabilities with reduced validation or different tool permissions. Fallback agents designed for resilience often operate with relaxed constraints to ensure minimum viable functionality during degraded operation, creating capability bypass through deliberate failure injection. In multi-agent systems with shared fallback pools, forcing primary failures across multiple agents simultaneously routes the entire fleet to shared fallback agents with reduced oversight.
