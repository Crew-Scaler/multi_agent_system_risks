# 13.5.7 Kubernetes Secret Store Poisoning for Shared Agent Credentials [NoFW]

Kubernetes secrets storing NGC API keys and other shared credentials (used by all NIM/agent pods) can be poisoned if attackers gain write access to the secret store (etcd). Compromising one shared secret affects all agent services authenticating through it. Unlike singular deployments with independent credentials, multi-agent Kubernetes deployments using shared secrets enable single-secret-poisoning attacks propagating to all agents simultaneously, converting centralized credential management from an operational convenience into a single point of fleet-wide failure.

## Supporting Evidence

- Saurabh Deochake, Ryan Murphy, and Jeremiah Gearheart. 2025. A Multi-Cloud Framework for Zero-Trust Workload Authentication. In *Cyber Security Experimentation and Test (CSET) Workshop at ACSAC 2025*. https://doi.org/10.1109/acsacw69556.2025.00061
  - [→ 2510.16067.md](../../papers/2510.16067.md)
  - "A single compromised key can function as a 'skeleton key' to vast cloud resources, enabling a clear threat model where an adversary's primary goals are data exfiltration, resource hijacking, and lateral movement between cloud environments"
  - "the lack of robust, automated rotation and revocation mechanisms leads to credential sprawl, where keys with overly broad permissions dramatically increase the potential blast radius of a single compromise"
