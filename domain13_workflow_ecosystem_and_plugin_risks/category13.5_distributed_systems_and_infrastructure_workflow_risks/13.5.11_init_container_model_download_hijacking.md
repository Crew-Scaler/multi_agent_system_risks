# 13.5.11 Init Container Model Download Hijacking [NoFW]

Kubernetes init containers downloading model weights from external registries (NGC, HuggingFace, S3) before main agent containers start are vulnerable to supply chain attacks at the download layer. Attackers performing man-in-the-middle attacks on model download connections, poisoning DNS to redirect downloads to attacker-controlled endpoints, or compromising external registry contents cause init containers to download adversarial model weights before the main agent container's security controls activate. In multi-agent Kubernetes clusters, all pods using the same init container specification download from the same endpoint, enabling one supply chain compromise to affect all cluster deployments simultaneously.
