# 13.5.22 Production Vector Database Authentication Bypass Through Shared API Key Reuse [NoFW]

Production vector database deployments use API key authentication, but multi-agent systems often share single API keys across all agents for operational simplicity. Attackers compromising one agent obtain the shared API key, gaining unauthorized access to the vector database used by the entire agent fleet. Single-agent systems can use unique API keys per deployment; multi-agent shared vector databases with single authentication credentials create 1-to-N compromise vectors affecting all agents' retrieval pipelines simultaneously.
