# 13.5.12 Message Queue Idempotency Violation Attacks Through Duplicate Message Injection [NoFW]

Message queue systems implement idempotency keys to prevent duplicate processing of retried messages. Attackers inject duplicate messages with identical idempotency keys to legitimate messages, exploiting race conditions in idempotency checking where duplicate messages are processed before idempotency records are committed. In multi-agent systems where multiple agents consume from shared queues, duplicate processing can trigger multiple agent executions of the same workflow step, causing unintended repeated side effects — duplicate financial transactions, repeated tool invocations, or conflicting state updates.
