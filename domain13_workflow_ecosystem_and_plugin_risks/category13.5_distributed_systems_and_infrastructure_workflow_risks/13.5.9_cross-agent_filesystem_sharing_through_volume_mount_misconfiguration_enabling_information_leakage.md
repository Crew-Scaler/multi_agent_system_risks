# 13.5.9 Cross-Agent Filesystem Sharing Through Volume Mount Misconfiguration Enabling Information Leakage [NoFW]

Production containerized multi-agent deployments using shared Kubernetes PersistentVolumeClaims with misconfigured filesystem permissions (default 755 instead of 700, or all containers running as the same UID) enable any agent container to read and write any other agent's directory. Attackers compromising one lower-privilege agent can scan all 45 agent directories in 30 seconds, harvesting cached database credentials from 40% of the fleet and achieving fleet-wide lateral movement. Malicious file injection via shared volume access can backdoor all agent startup scripts, persisting through container restarts and Kubernetes rolling updates. Single-agent dedicated PVCs eliminate this cross-agent attack surface entirely.
