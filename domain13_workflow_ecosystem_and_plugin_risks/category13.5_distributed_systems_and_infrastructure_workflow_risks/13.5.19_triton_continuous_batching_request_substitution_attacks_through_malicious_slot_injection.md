# 13.5.19 Triton Continuous Batching Request Substitution Attacks Through Malicious Slot Injection [NoFW]

Triton Inference Server's continuous batching dynamically allocates inference compute slots to incoming requests, enabling attackers to inject carefully timed requests that claim compute slots during high-load periods when slot allocation mechanisms operate under degraded validation. Attacker-controlled requests occupying critical inference slots can influence shared batch-level computations through co-batch attention mechanisms, substituting attacker-controlled inputs into inference slots that victim requests expect to occupy exclusively. In multi-agent Triton deployments, slot injection attacks exploit shared inference infrastructure affecting all agents whose requests are batched in the same continuous batching window.
