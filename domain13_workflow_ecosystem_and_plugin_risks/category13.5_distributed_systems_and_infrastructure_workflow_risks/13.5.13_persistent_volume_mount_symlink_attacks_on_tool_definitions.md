# 13.5.13 Persistent Volume Mount Symlink Attacks on Tool Definitions [NoFW]

Container-based agent deployments mounting tool definition files from shared persistent volumes are vulnerable to symlink attacks when container runtime configurations allow symlink following across volume boundaries. Attackers create adversarial symlinks within mounted volumes pointing to sensitive host filesystem paths or other containers' private directories, causing agent containers following symlinks to read attacker-specified file contents as tool definitions. In multi-agent deployments sharing PVC-mounted tool definition directories, one symlink attack affects all containers with that volume mounted simultaneously.
