# 13.5.10 Model Cache Poisoning Through Shared Persistent Volumes [NoFW]

Multi-agent Kubernetes deployments mounting shared persistent volumes for model weight caching enable cross-agent model cache poisoning. Attackers with write access to shared volume mounts replace legitimate model weight files with adversarially modified versions, causing all agents loading from the shared volume to use poisoned weights. Unlike per-pod model storage where poisoning is limited to one deployment, shared PVC-based model caches create single-volume-compromise scenarios propagating to all agents in the same PVC access group simultaneously.
