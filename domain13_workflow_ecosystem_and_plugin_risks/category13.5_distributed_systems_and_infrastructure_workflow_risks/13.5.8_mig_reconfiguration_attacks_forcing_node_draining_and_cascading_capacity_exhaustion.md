# 13.5.8 MIG Reconfiguration Attacks Forcing Node Draining and Cascading Capacity Exhaustion [NoFW]

Adversaries with Kubernetes API access exploit MIG (Multi-Instance GPU) reconfiguration workflows to force cascading capacity exhaustion by triggering unnecessary reconfigurations across multiple GPUs simultaneously, draining nodes faster than the cluster can reschedule pods and saturating remaining capacity. Attacks timed during peak hours (85% baseline utilization) push utilization above 97% causing latency spikes, while mixed profile fragmentation strands 30-40% of idle capacity in wrong partition sizes creating artificial scarcity. Single-agent deployments on dedicated GPU infrastructure contain MIG changes to one deployment; multi-agent shared clusters create cascading failures affecting the entire fleet.
