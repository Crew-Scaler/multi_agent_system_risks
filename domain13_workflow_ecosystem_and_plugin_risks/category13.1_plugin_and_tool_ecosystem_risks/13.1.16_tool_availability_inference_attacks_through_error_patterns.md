# 13.1.16 Tool Availability Inference Attacks Through Error Patterns [NoFW]

Different tool error codes and response patterns reveal internal system state — which tools exist, which are operational, which permissions are active — allowing attackers to map the tool ecosystem through systematic probing. By inducing specific error conditions and observing error message content, error codes, and response timing, attackers construct detailed maps of available tools, their parameter requirements, and their access control configurations. In multi-agent systems, cross-agent error pattern analysis reveals tool availability across the entire fleet, enabling precision targeting of vulnerable tool endpoints.
