# 13.1.17 Precondition Inference Attack Through Tool Failure Analysis [NoFW]

Tools with hidden preconditions (authentication state, prior action requirements, resource availability) reveal those preconditions through failure messages when preconditions are unmet. Attackers systematically invoke tools under various states, analyzing failure responses to reverse-engineer precondition requirements that are not documented in tool schemas. Once preconditions are mapped, attackers craft sequences satisfying all preconditions for high-privilege tool invocations that nominal workflows do not reach. In multi-agent systems, coordinated precondition mapping across multiple agents accelerates reverse engineering of complex multi-step precondition chains.
