# 13.1.18 Tool Schema Ambiguity in Parameter Extraction [NoFW]

Tool schemas with ambiguous parameter specifications (overlapping enum values, underspecified type constraints, ambiguous field names) are interpreted inconsistently across different agent implementations and LLM models. Attackers craft inputs exploiting schema interpretation differences between agents — parameters interpreted as safe by one agent's parser are interpreted as malicious by another agent's execution layer. Multi-agent systems using heterogeneous LLM backends create cross-agent semantic gaps that enable adversarial inputs to pass validation in one agent while producing harmful behavior when re-processed by a different agent.
