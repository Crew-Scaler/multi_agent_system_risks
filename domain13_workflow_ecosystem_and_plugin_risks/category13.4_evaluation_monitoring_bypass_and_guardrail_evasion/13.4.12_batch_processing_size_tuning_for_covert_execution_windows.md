# 13.4.12 Batch Processing Size Tuning for Covert Execution Windows [NoFW]

Batch processing systems accumulate operations before executing them as groups, creating time windows between batch composition and batch execution during which individual operations are not monitored in real-time. Attackers tune batch composition timing to maximize the window between operation submission and execution visibility, embedding adversarial operations within large legitimate batches where they are less likely to trigger per-operation monitoring alerts. In multi-agent systems sharing batch processing infrastructure, one agent's batch manipulation affects all agents' operation visibility simultaneously through shared batching queues.
