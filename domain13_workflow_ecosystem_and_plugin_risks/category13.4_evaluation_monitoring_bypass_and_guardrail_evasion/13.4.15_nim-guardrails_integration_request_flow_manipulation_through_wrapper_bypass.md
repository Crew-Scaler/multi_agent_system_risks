# 13.4.15 NIM-Guardrails Integration Request Flow Manipulation Through Wrapper Bypass [NoFW]

NVIDIA NIM deployments implementing guardrails through wrapper layers that intercept and validate requests before forwarding to inference endpoints are exploited by crafting requests that pass wrapper validation while containing payloads that bypass guardrail logic when processed by the underlying inference engine. Attackers exploit semantic gaps between wrapper-level validation (operating on request structure and metadata) and engine-level inference (operating on full token sequences), submitting requests validated as safe by the wrapper that produce unsafe outputs from the inference engine. In multi-agent NIM deployments, wrapper bypass techniques effective against one NIM endpoint apply to all NIM endpoints using the same guardrail configuration.
