# 9.5.4 Error Recovery Payload Persistence Through Retry Cycles [NoFW]

Error recovery implementing automatic retry creates attack opportunities where malicious payloads stored in error state persist across retry cycles. Attackers craft errors generating malicious observations stored in memory as context from prior attempts, remaining in agent memory through multiple retries. Each retry retrieves this poisoned memory as legitimate prior attempt state, activating malicious instructions through memory reuse across error recovery cycles in multi-agent systems where error context is shared.

- Chen, Zhaorun, Guo, Zhen, Xiao, Chaowei, Li, Bo, and Zhao, Ruoxi. 2024. AgentPoison: Red-teaming LLM Agents via Poisoning Memory or Knowledge Bases. In *NeurIPS 2024*. https://doi.org/10.52202/079017-4136
