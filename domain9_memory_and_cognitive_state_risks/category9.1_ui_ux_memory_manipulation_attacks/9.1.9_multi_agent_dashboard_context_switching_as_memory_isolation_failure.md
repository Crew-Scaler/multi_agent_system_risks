# 9.1.9 Multi-Agent Dashboard Context Switching as Memory Isolation Failure [NoFW]

Multi-agent dashboard UIs displaying parallel conversation threads with different agents create memory isolation expectations that fail when agents share underlying session state or context stores. Users expect cognitive isolation between separate chat panels, but when agents share session persistence, conversation history, or learned user preferences through backend context stores, memory poisoning in one agent context contaminates other contexts invisibly. The visual metaphor of separate panels misrepresents actual cognitive boundaries, enabling memory poisoning attacks that are invisible to users observing UI-level isolation.
