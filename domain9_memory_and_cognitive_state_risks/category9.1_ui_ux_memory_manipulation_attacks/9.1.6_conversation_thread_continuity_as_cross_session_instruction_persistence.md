# 9.1.6 Conversation Thread Continuity as Cross-Session Instruction Persistence [NoFW]

Conversation thread continuity features maintaining coherent multi-turn dialogues across session boundaries enable cross-session instruction persistence where malicious payloads injected in Session 1 remain active in Session N days or weeks later. In multi-agent systems, conversation threads span multiple agent transitions, with each agent adding to persistent thread based on specialized processing. Attackers inject malicious instructions disguised as clarifying questions early, which become part of the conversation's semantic context that later agents reference. Multi-agent vulnerability is amplified because threads persist across agent specialization boundaries—poisoning during low-privilege research agent interaction affects high-privilege execution agents referencing the same history.
