# 9.1.11 Context Window Limit Warnings as Memory Eviction Attack Vectors [NoFW]

Context awareness features displaying warnings when the agent's context window approaches limits create memory eviction attack vectors where attackers force eviction of security-relevant context while preserving poisoned instructions. Multi-agent systems with agents having different window sizes and eviction policies create opportunities where malicious instructions with injected high-priority markers are preserved while security constraints get evicted. The UI pattern of automatic context management never prompts explicit preservation choices, and multi-agent desynchronization means one agent evicts a "verify credentials" constraint while another retains it, creating exploitable security posture inconsistencies the UI's unified display obscures.
