# 9.7.4 Cached Embedding Corruption in Shared Vector Stores [NoFW]

Multi-agent RAG systems caching embeddings in shared vector stores to avoid recomputation create poisoning targets when multiple agents trust the cache without validating embedding integrity. Attackers poison the cache by injecting embeddings computed with compromised models, poisoned input text, or deliberately mismatched dimensions. Once malicious embeddings enter the shared cache, all agents retrieving from that cache incorporate poisoned vectors into their RAG pipelines, and the corruption is invisible to agents inspecting retrieved text since vectors appear numerically valid but represent adversarially crafted representations.
