# 9.7.5 Persistent Volume Poisoning in Vector Database Storage Layers [NoFW]

Production vector databases storing HNSW graph structures, embeddings, and metadata in persistent volumes create targets for attackers with volume access. Unlike in-memory poisoning affecting only current sessions, volume poisoning directly modifies stored graph files and vector data, corrupting durable storage permanently until volume restoration. Attackers can introduce malicious edges in graph files, alter distance values to favor attacker-controlled documents in rankings, corrupt embedding values to shift semantic meanings, or modify metadata for document substitution. All downstream agents querying the database receive poisoned data until a clean restore is performed.
