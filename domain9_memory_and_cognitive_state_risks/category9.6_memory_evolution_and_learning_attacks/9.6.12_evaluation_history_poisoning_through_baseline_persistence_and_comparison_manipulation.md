# 9.6.12 Evaluation History Poisoning Through Baseline Persistence and Comparison Manipulation [NoFW]

Evaluation pipelines maintaining historical baselines for trend analysis create persistent poisoning targets in multi-agent evaluation systems. Attackers poison historical data through baseline cache poisoning (modifying stored metrics to establish false reference points) or continuous evaluation baseline degradation (altering records to make current performance appear better). Compromising one shared baseline database affects all downstream evaluation agents relying on those baselines, enabling long-term evaluation bias through persistence.
