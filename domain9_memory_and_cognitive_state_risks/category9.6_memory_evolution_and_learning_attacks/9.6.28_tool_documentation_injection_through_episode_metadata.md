# 9.6.28 Tool Documentation Injection Through Episode Metadata [NoFW]

Episodic memory systems storing agent interaction histories for retrieval include tool invocation metadata. Attackers craft tool invocations whose episode metadata fields (descriptions, outcome labels, context tags) contain instruction-injection payloads. When subsequent agents retrieve these episodes as examples of successful tool use, the injected metadata becomes part of the retrieved context, embedding attacker instructions into the agent's operational context through legitimate episodic memory retrieval. Unlike direct prompt injection, this attack persists across sessions as a memory store entry.

  - "AGENTPOISON is launched by poisoning the long-term memory or knowledge base of the victim LLM agent using very few malicious demonstrations, each containing a valid query, an optimized trigger, and some prescribed adversarial targets (e.g., a dangerous sudden stop action for autonomous driving agents)."
  - "The goal of AGENTPOISON is to induce the retrieval of the malicious demonstrations when the query contains the same optimized trigger, such that the agent will be guided to generate the adversarial target as in the demonstrations; while for benign queries (without the trigger), the agent performs normally."
