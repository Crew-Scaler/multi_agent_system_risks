# 9.8.10 MinHash Fuzzy Deduplication Hash Collision Injection Creating Training Data Duplication Amplification [NoFW]

MinHash probabilistic deduplication used for large-scale corpus processing exhibits hash collision vulnerabilities where adversaries craft documents producing identical signatures to legitimate content despite semantic differences. This enables adversarial content to either survive deduplication while displacing legitimate documents (hash collision causing legitimate content to be marked duplicate and discarded) or to be amplified through failed deduplication. Multi-agent centralized deduplication creates fleet-wide training poisoning where signature collisions affect all agents trained on the corpus, enabling coordinated training data substitution across multi-agent deployments through fuzzy deduplication manipulation.
