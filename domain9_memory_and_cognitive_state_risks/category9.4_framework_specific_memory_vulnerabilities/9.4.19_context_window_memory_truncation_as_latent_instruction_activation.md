# 9.4.19 Context Window Memory Truncation as Latent Instruction Activation [NoFW]

Multi-agent systems tuning context windows differently per agent (varying token limits) create truncation points where memory gets summarized or compressed. Attackers craft instructions designed to activate during memory truncation—instructions that survive summarization logic while benign security constraints are evicted. Attackers with knowledge of per-agent context window configurations can craft instructions persistent under specific truncation regimes, a vulnerability unique to multi-agent systems with heterogeneous window sizes.
