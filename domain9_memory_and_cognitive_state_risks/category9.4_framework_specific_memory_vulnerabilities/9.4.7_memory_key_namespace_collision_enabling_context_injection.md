# 9.4.7 Memory Key Namespace Collision Enabling Context Injection [LangChain]

LangChain memory systems use memory_key parameters to inject context into prompts, creating vulnerability when multiple agents share memory stores using identical key names. Attackers exploit namespace collisions by injecting malicious content under shared keys, causing all agents referencing those keys to incorporate poisoned context. Multi-agent architectures standardizing memory key names for convenience create collective vulnerabilities absent from isolated singular agent systems with separate memory namespaces.
