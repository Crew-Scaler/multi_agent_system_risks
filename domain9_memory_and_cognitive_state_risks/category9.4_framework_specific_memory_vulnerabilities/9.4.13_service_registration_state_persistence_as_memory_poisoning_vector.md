# 9.4.13 Service Registration State Persistence as Memory Poisoning Vector [Semantic Kernel]

Kernel service registrations persist for the lifetime of the kernel instance. Attackers manipulating service registration state by de-registering legitimate services or registering malicious replacements cause corruption persisting across all subsequent plugin invocations. Multi-agent systems where plugins coordinate through shared kernel services face backdoors activating through service mutation, affecting all downstream agents in contrast to federated per-agent kernels that would isolate corruption.
