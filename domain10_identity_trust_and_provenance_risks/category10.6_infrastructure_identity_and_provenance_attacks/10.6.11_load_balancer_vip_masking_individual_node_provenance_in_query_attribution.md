# 10.6.11 Load Balancer VIP Masking Individual Node Provenance in Query Attribution [NoFW]

Clustered vector database deployments use load balancers with virtual IP addresses that mask individual node identities, creating provenance gaps where audit logs cannot determine which specific node processed which agent's request. Attackers exploit this by compromising one node knowing that audit logs won't reveal the node identity, enabling persistent data exfiltration with attribution confusion across all agents sharing the load balancer.
