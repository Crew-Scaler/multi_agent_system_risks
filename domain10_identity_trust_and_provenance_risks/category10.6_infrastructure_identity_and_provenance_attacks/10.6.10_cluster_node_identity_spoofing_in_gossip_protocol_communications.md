# 10.6.10 Cluster Node Identity Spoofing in Gossip Protocol Communications [NoFW]

Multi-node vector database clusters use gossip protocols for membership coordination identified by CLUSTER_HOSTNAME values. Attackers spoof node identities by forging CLUSTER_HOSTNAME values in gossip messages, creating identity conflicts that enable data poisoning, query interception, and consensus disruption. Multi-agent systems querying clusters trust cluster topology without verifying individual node identity, amplifying this risk.
