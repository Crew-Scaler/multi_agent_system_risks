# 11.1.11 Multi-Agent Framework Update Non-Determinism Invalidating Cumulative Assurance [NoFW]

AutoGen, CrewAI, and underlying LLMs update on different schedules. Evaluating N agents across M framework versions across P LLM versions creates N×M×P re-testing combinations that make comprehensive cumulative assurance infeasible.

  - "This instability lowers confidence in the reliability of reporting only a single number in LLM benchmarks. We encourage reporting maximum-minimum scores across runs to have a more robust comparison of models."

  - "even when performing random sampling with multiple independent runs, the averaged results can still vary significantly due to hardware and system-level nondeterminism."
  - "When results cannot be exactly reproduced, it becomes difficult to distinguish whether improvements are from better methods or merely random variation."
