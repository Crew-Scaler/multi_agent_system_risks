# 11.1.7 Tool Availability Changes Creating Non-Deterministic Behavior [NoFW]

Tools deprecate and capabilities change post-deployment. Multi-agent systems have agents using outdated tool lists while others use updated lists, creating state where older agents invoke deprecated tools that newer agents have removed, invalidating assurance across the fleet.

## Supporting Evidence

- Jehyeok Yeon, Isha Chaudhary, and Gagandeep Singh. 2026. Quantitative Certification of Agentic Tool Selection. In *ICLR 2026 Workshop on Agents in the Wild: Safety, Security, and Beyond*. https://arxiv.org/abs/2510.03992
  - [→ 2510.03992.md](../../papers/2510.03992.md)
  - "a pipeline's behavior in deployment depends on the tool pool the agent actually encounters, which in open registries is shaped by third parties"
  - "The certificate is conditional on a fixed pool. Substantive pool changes require re-evaluation."
- Stephan Rabanser, Sayash Kapoor, Peter Kirgis, Kangheng Liu, Saiteja Utpala, and Arvind Narayanan. 2026. Towards a Science of AI Agent Reliability. In *ICML 2026*. https://arxiv.org/abs/2602.16666
