# Category 4.7 [NEW]: Streaming and Token-Level Leakage [RDL_3, RDL_10, RDL_26]

*Alignment: Expands GenAI Domain 4 with four new categories covering large-context, log-based, streaming, and probabilistic recall leakage vectors unique to MAS architectures (RDL domain, 36 categories).*

**Description:** Streaming token generation and real-time agent output processing create time-windowed leakage opportunities where partial outputs, token timing patterns, and intermediate reasoning states expose sensitive information before completion.

## Key Risks

- 4.7.1 Streaming Response Token-by-Token Data Leakage: Sensitive data included in early streaming tokens is transmitted and potentially logged before the agent's safety filtering or content review of the complete response. [NoFW]
- 4.7.2 Streaming Output Length Correlation for Session Reconstruction: Observers measure response lengths and token timing patterns to infer the content and scope of agent tasks without accessing message content. [NoFW]
- 4.7.3 CoT Trace Leakage Through Memory Queries: Chain-of-thought reasoning traces persisted to shared memory expose the agent's intermediate reasoning, embedded data, and decision rationale to other agents and system components. [NoFW]
- 4.7.4 KV Cache Side-Channel Leakage Through Tensor Parallelism: Shared GPU tensor parallelism creates cross-user leakage opportunities through observable differences in cache behavior between requests. [NoFW]
- 4.7.5 Streaming State Updates in Multi-Agent Cycles Leaking Progressive State: Intermediate state updates broadcast between agents during multi-step workflows expose partial results before the workflow reaches a privacy-safe completion state. [NoFW]
- 4.7.6 Streaming Response Capture Creating Data Exfiltration Windows: LangChain agents with streaming enabled generate output token-by-token, creating transient windows where sensitive information appears in network streams, browser buffers, and logs before post-processing redaction applies. Multi-agent streaming handoffs — Agent A streams to Agent B which streams to user — create multiple data leakage windows, unlike singular streaming that exposes data at a single interface point. [NoFW]
- 4.7.7 Streaming Tokenization Enabling Context Window Inference Attacks: Streaming reveals how content is tokenized — token boundaries, special tokens, embedding dimensions — enabling inference attacks on context windows. In multi-agent systems, attackers observing streaming patterns from multiple agents can infer context window sizes, token allocations, and information prioritization schemes, enabling attackers to map agent capabilities through streaming patterns in ways that batch execution cannot reveal. [NoFW]
- 4.7.8 Streaming Response Intermediate States as Information Leakage: Streaming responses in batched or load-balanced contexts reveal intermediate processing states where partial responses show partial results enabling attackers to infer computation progress. Different agents' streaming patterns reveal which agent is processing which request, enabling observation of intermediate computation that reveals agent identities and workload distribution — information that complete batch responses conceal. [NoFW]
- 4.7.9 Streaming Response Caching for Covert Data Exfiltration: Streaming responses cached for efficiency create opportunities for attackers exfiltrating data through cache side channels. Multi-agent systems caching streamed responses enable attackers querying caches to determine what streams were generated, potentially recovering sensitive information from content cached during agent-to-agent handoffs. The N-party visibility into streaming content in multi-agent cache sharing exceeds what any singular system exposes. [NoFW]
- 4.7.10 Session Affinity Persistence Revealing Agent Specialization: IP hash load balancing creates persistent sessions enabling attackers to observe agent specialization through request and response patterns over time. By analyzing a single long-lived session, attackers can infer what capabilities that agent possesses and profile agent specialization. Persistent session affinity transforms transient behavioral observations into durable agent capability profiles, enabling targeted data extraction attacks against the most information-rich agents. [NoFW]
- 4.7.11 Engine Binary Metadata Leakage: TensorRT engine binaries contain metadata including model architecture, layer configurations, and optimization decisions. Attackers with access to deployed engines can extract architectural information enabling model extraction attacks. In multi-agent systems where multiple agents execute engines built from the same base model, extracting one agent's engine enables inferring all agents' architectures through architectural similarity analysis, creating amplified leakage absent in single-agent deployments. [NoFW]
- 4.7.12 Quantization Artifacts as Probabilistic Leakage Channels: INT8 quantization introduces systematic artifacts in token probabilities where certain token ranges are quantized more aggressively, enabling attackers analyzing model outputs to infer which activation ranges were exercised during inference and thus reveal information about input content. Multi-agent systems where agents process different data types create multi-domain leakage channels where quantization artifacts from each specialized agent leak domain-specific activation patterns. [NoFW]
- 4.7.13 Calibration Dataset Artifacts in Quantization Statistics: INT8 quantization calibration produces statistics — min/max activation ranges, histogram data — stored with engine metadata that leak information about the calibration dataset distribution. Specialized agents with domain-specific calibration expose their operational domain through these statistics, enabling attackers to select attack targets based on revealed operational domains. Multi-agent deployments with N specialized agents create N statistic sets each revealing a different calibration domain. [NoFW]
