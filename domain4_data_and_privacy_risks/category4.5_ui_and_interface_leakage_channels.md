# Category 4.5 [NEW]: UI and Interface Leakage Channels [RDL_1, RDL_2, RDL_8]

*Alignment: Expands GenAI Domain 4 with four new categories covering large-context, log-based, streaming, and probabilistic recall leakage vectors unique to MAS architectures (RDL domain, 36 categories).*

**Description:** Agent user interfaces — including chat history displays, progressive disclosure panels, copy-paste functions, and approval workflow logs — create leakage channels that expose sensitive operational data through mechanisms not present in conventional software systems.

## Key Risks

- 4.5.1 Chat Interface Conversation History as Persistent Sensitive Data Repository: Chat interfaces retain conversation history containing tool parameters, intermediate reasoning, and data processed by agents, creating a persistent exposure surface. [NoFW]
- 4.5.2 Progressive Disclosure Technical Views as Secret Exposure Vectors: Expanded technical views in agent dashboards reveal implementation details, internal identifiers, and operational parameters not visible in standard views. [NoFW]
- 4.5.3 Context Awareness Session Persistence Across Security Boundaries: Agent context persisting across session boundaries exposes data from one security context to users or agents operating in a different security context. [NoFW]
- 4.5.4 Copy-Paste from Chat Interfaces Transferring Embedded Metadata: Clipboard data transferred from agent chat interfaces includes hidden metadata (timestamps, agent IDs, confidence scores) alongside visible text. [NoFW]
- 4.5.5 Time-Boxed Approval Defaults Logging Automated Decisions Without Attribution: Approval workflow logs record automated timeout-triggered decisions as human-approved actions, creating false attribution in audit records. [NoFW]
- 4.5.6 Command Palette Recent History as Sensitive Operation Log: Command palette patterns displaying recently executed commands create logs of sensitive user operations persisting in UI memory, browser storage, and palette suggestion databases. Multi-agent command palettes record commands across all integrated agents, showing not just what users did but cross-agent operation sequences — accessed HR database, then finance database, then sent email — indicating potential insider threat patterns. Unlike application-specific histories revealing one tool's usage, multi-agent palettes document comprehensive user behavior across entire ecosystems including sequences revealing high-level intent. [NoFW]
- 4.5.7 Multi-Agent Dashboard Aggregated Context Display: Multi-agent dashboard UIs displaying aggregated status, outputs, and reasoning from multiple agents simultaneously create comprehensive views exposing correlations between sensitive data sources intended to remain compartmentalized. Dashboards can simultaneously show customer data from CRM agents, financial projections from accounting agents, employee information from HR agents, and security alerts — correlations revealing business intelligence or privacy violations from data that policy intends to be segregated. Unlike role-based views in traditional applications, multi-agent dashboards aggregate complete system state across security boundaries in a single view. [NoFW]
- 4.5.8 Context Reference Links as Indirect Information Disclosure: Context awareness features creating clickable reference links connecting current responses to earlier context expose sensitive information indirectly by revealing discussion topics, when discussed, and relationships between them. Multi-agent context reference graphs span multiple agents' contexts, exposing information flows through the entire multi-agent system. Attackers with conversation access can reconstruct sensitive business context by following reference links even if individual messages are partially redacted — the link structure itself reveals correlations showing how information relates across the organization. [NoFW]
- 4.5.9 Inline Suggestion Context Leakage Through Autocomplete: Inline suggestion patterns using agent intelligence to autocomplete inputs leak information about agent predictions based on current context. Multi-agent suggestion systems aggregate predictions from multiple agents — code agents suggest function names based on codebase, documentation agents suggest product names, customer agents suggest client names — revealing what data agents access and what patterns they have learned. Unlike manual entry leaking only explicit typing, agent suggestions proactively leak backend data and learned patterns as side channels where observers infer sensitive information from the autocomplete suggestions rather than from explicit requests. [NoFW]
- 4.5.10 Approval Workflow Audit Trails as Comprehensive Behavior Logs: Approval workflow UIs maintaining detailed audit trails capture every decision request, evidence examined, confidence score calculated, and human review comment — comprehensive records that become high-value reconnaissance targets. Multi-agent approval workflows aggregate sensitive business logic from multiple agents, effectively documenting entire organizational decision-making including pricing algorithms, fraud detection patterns, and regulatory edge cases in queryable log databases. Unlike singular-agent audit logs revealing one component's reasoning, multi-agent approval trails document entire distributed decision-making architecture, making them comprehensive intelligence sources for adversaries. [NoFW]
- 4.5.11 Approval Workflow Supporting Evidence Links as Data Aggregation Points: Approval workflow UIs providing direct links to supporting evidence create concentrated data aggregation points where evidence links provide access to sensitive source materials potentially more sensitive than the approval decision itself. Multi-agent workflows aggregate evidence from diverse backend systems accessed by different agents spanning multiple security domains without consistently applying access controls. Users reviewing approvals may click evidence links gaining access to raw data their roles should not permit, effectively creating indirect access paths bypassing normal controls. [NoFW]
- 4.5.12 Interactive Refinement Iteration History as Decision Logic Disclosure: Collaborative patterns supporting interactive refinement through multiple iterations create detailed histories exposing both agent decision-making and user refinement logic. Multi-agent refinements involve multiple specialized agents — Draft, Review, Refinement, Quality — with complete iteration histories revealing how each agent evaluates quality, what patterns trigger revisions, and finalization criteria. Attackers analyzing these histories reverse-engineer decision logic enabling adversarial attacks that craft inputs to manipulate iteration behavior or bypass quality gates. [NoFW]
- 4.5.13 Sensitive Data Exposure Through Alt Text and ARIA Labels: Accessibility features supporting screen readers inadvertently expose sensitive data extractable via DOM scraping. Alt text and ARIA labels in financial chart images may contain precise revenue figures while ARIA labels for masked fields may contain full unmasked values for screen reader accessibility. Multi-agent contexts amplify this because one agent's accessible output becomes another's input — alt text generated for accessibility may expose details when consumed downstream — and automated tools can extract all attributes bypassing visual controls. [NoFW]
