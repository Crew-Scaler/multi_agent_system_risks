# Category 4.6 [NEW]: Data Persistence and Caching Leakage [RDL_2, RDL_5, RDL_6]

*Alignment: Expands GenAI Domain 4 with four new categories covering large-context, log-based, streaming, and probabilistic recall leakage vectors unique to MAS architectures (RDL domain, 36 categories).*

**Description:** Agent caching systems, session serialization, and framework-specific logging create persistent stores of sensitive operational data that outlast their intended scope and can be accessed through timing, log analysis, or side-channel observation.

## Key Risks

- 4.6.1 Cached Responses Persisting Sensitive Data Beyond Session Lifetime: Shared caching infrastructure retains agent responses containing sensitive data after sessions end, exposing that data to subsequent queries that hit the same cache entries. [NoFW]
- 4.6.2 Framework-Dependent Logging Enabling Data Leakage Through Debug Transparency: Frameworks like AutoGen and CrewAI log detailed execution traces (inter-agent messages, task decisions) that contain sensitive operational data when debug logging is enabled in production. [NoFW]
- 4.6.3 Multi-Agent Attribution Logs as Organizational Intelligence: Aggregated attribution logs revealing which agents collaborate, when, and on what tasks expose organizational workflow patterns and priorities to attackers. [NoFW]
- 4.6.4 Latency Measurement Information Leakage: Systematic latency differences between agent operations reveal computational load, tool availability, and data volume to observers without access to content. [NoFW]
- 4.6.5 Undo Functionality Preserving Deleted Sensitive Data: Agent interfaces with undo capabilities maintain state buffers containing data the user intended to delete, creating recoverable sensitive data stores. [NoFW]
- 4.6.6 Session Persistence Serialization as Structured Data Exfiltration: Context awareness session persistence converting unstructured conversation into structured queryable formats enables efficient sensitive data extraction. Multi-agent serialized sessions contain structured representations of all agent interactions, decisions, data accesses, and reasoning traces including metadata like timestamps, identifiers, confidence scores, and lineage not appearing in the UI but captured for restoration. Attackers compromising session storage databases can efficiently extract using structured queries across thousands of sessions, transforming what were text logs into queryable databases enabling large-scale exfiltration operations. [NoFW]
- 4.6.7 AutoGen Conversation History Aggregating Sensitive Inter-Agent Communication: AutoGen's shared GroupChat message history captures complete inter-agent conversations including negotiations, corrections, and reasoning exchanges between agents not intended for human exposure. The history aggregates sensitive data from all agents' internal communications including agent-to-agent debates about decisions, disagreements, and alternative approaches that reveal decision-making logic and organizational vulnerabilities. Unlike singular agent systems that produce no inter-agent communication logs, AutoGen's shared history creates a new leakage channel combining all agents' exchanges in a unified queryable log. [NoFW]
- 4.6.8 Checkpoint Storage Leaking Complete Workflow Context: LangGraph checkpoints persist entire state including conversation history, tool outputs, intermediate results, and agent reasoning, storing comprehensive snapshots in databases accessible to multiple agents and potentially exposed in backups. Checkpoints may include sensitive data flowing through the workflow from finance, HR, and security agents within the same state, creating high-value reconnaissance targets that aggregate data from multiple specialized agents' domains. Unlike single-agent checkpoint leakage involving one agent's data, multi-agent checkpoints aggregate multiple agents' contexts creating comprehensive organizational intelligence. [NoFW]
- 4.6.9 Conversation History Data Leakage Through Memory Persistence: LangChain's ConversationBufferMemory persists full conversation history often stored in databases or files without encryption, creating data leakage risks where tool outputs, intermediate analyses, and reasoning traces containing sensitive data persist in plaintext. Multi-agent systems sharing conversation history through centralized memory stores aggregate sensitive data from all agents into single high-value targets, unlike singular agent memory which remains agent-local. [NoFW]
- 4.6.10 Memory Serialization Leaking Internal Representation: ConversationBufferMemory serializes conversation history to storage including metadata such as timestamps, internal IDs, and token counts that provide attackers with additional intelligence about agent behavior patterns and timing. Multi-agent memory serialization includes cross-agent coordination data showing which agents communicated when, providing architectural reconnaissance that single-agent serialization cannot produce. [NoFW]
- 4.6.11 Framework-Specific State Serialization Enabling Probabilistic Reconstruction Attacks: Different frameworks serialize state differently for persistence — LangChain's JSON conversation history, LangGraph's TypedDict serialization, AutoGen's message object serialization, CrewAI's task context encoding. Multi-agent systems storing heterogeneous serialized states in shared databases enable probabilistic reconstruction attacks where attackers analyze serialization patterns, detecting sensitive data through format fingerprinting. Unlike singular systems with one serialization model, multi-agent systems expose multiple data encoding models enabling complementary attacks where patterns visible in one format guide reconstruction in another, revealing sensitive data probabilistically without direct content access. [NoFW]
