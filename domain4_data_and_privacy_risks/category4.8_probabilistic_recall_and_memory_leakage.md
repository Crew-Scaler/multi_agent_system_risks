# Category 4.8 [NEW]: Probabilistic Recall and Memory Leakage [RDL_4, RDL_31–36]

*Alignment: Expands GenAI Domain 4 with four new categories covering large-context, log-based, streaming, and probabilistic recall leakage vectors unique to MAS architectures (RDL domain, 36 categories).*

**Description:** Probabilistic retrieval, embedding similarity search, and memory consolidation in long-lived agent systems create leakage paths where sensitive historical data can be reconstructed or inferred through systematic querying even when direct access is blocked.

## Key Risks

- 4.8.1 Probabilistic Recall Through Conversation Search: Agents can be prompted to probabilistically recall sensitive data from past interactions through search-based memory retrieval even when direct access to that data is restricted. [NoFW]
- 4.8.2 Embedding Vector Inference as Probabilistic Recall Leakage: Systematic analysis of variance in repeated embedding retrievals reveals information about the underlying data used to generate those embeddings. [NoFW]
- 4.8.3 Semantic Memory Context Window Leakage Through Retrieved Documents: Documents retrieved from knowledge bases to satisfy agent queries expose content beyond what is directly relevant to the query. [NoFW]
- 4.8.4 Knowledge Graph Structure as Information Leakage Channel: Both present and absent relationships in agent knowledge graphs are information; systematic probing reveals organizational, operational, or personal associations. [NoFW]
- 4.8.5 Episodic Memory as Covert Data Exfiltration Mechanism: Attackers exploit episodic memory retrieval pathways to extract sensitive historical data embedded in prior agent interactions by crafting queries that trigger high-similarity retrieval. [NoFW]
- 4.8.6 Multi-Agent Conversation Trace Leakage Through Shared History: Shared conversation histories in multi-agent systems expose inter-agent communications and context from all participating agents to any agent that reads the shared history. [NoFW]
- 4.8.7 Intermediate Step Verbosity Enabling Data Reconstruction: CoT's explicit intermediate steps reveal the structure and content of intermediate computations. When these traces are logged or cached, attackers reconstruct sensitive inputs from step-by-step explanations — for example, inferring actual customer data from steps like "after filtering for age > 18, I have 450 records." Multi-agent systems logging CoT for coordination and replay preserve this step-by-step information that enables data reconstruction attacks, unlike single agents where intermediate steps persist only briefly. [NoFW]
- 4.8.8 Reasoning-Embedded Secrets in Shared Traces: Agents sometimes document reasoning about sensitive operations — for example, "I used API key [partial key shown] to verify credentials." These reasoning traces, stored for multi-agent coordination, expose secrets through documentation. Unlike single-agent reasoning that stays in ephemeral context, multi-agent shared reasoning is logged persistently, creating a new persistent repository of secrets embedded in seemingly innocent reasoning traces. [NoFW]
- 4.8.9 Cross-Agent Reasoning Context Leakage: When Agent A's reasoning trace is input to Agent B's reasoning process, all context that A considered — including sensitive data — flows to B. If B's reasoning is then logged or stored, the original sensitive context gets replicated across systems. Unlike single agents that process their own sensitive contexts, multi-agent systems where reasoning becomes input to other agents propagate sensitive context across agent boundaries without explicit data transfer controls. [NoFW]
- 4.8.10 Lookahead Computation Information Leakage via Tree State Observation: Tree-of-Thought agents perform lookahead by exploring future branches speculatively. In multi-agent systems, observing which branches are explored during lookahead can reveal future plans, strategies, or sensitive information. Unlike single-agent lookahead that remains internal, multi-agent systems expose lookahead decisions across agent boundaries, making planned but not-yet-executed actions observable through shared trace infrastructure. [NoFW]
- 4.8.11 Preserved Reasoning Path Context Leakage: Self-Consistency paths preserved for explanation purposes occupy large context windows and contain sensitive information from the original reasoning. In multi-agent systems where Agent A preserves paths containing sensitive data and downstream Agent B or monitoring agents access those paths, data leakage occurs through preserved context. Unlike single-agent preserved paths that remain within one context, multi-agent shared preserved path repositories create persistent leakage where multiple agents can access sensitive data from all agents' preserved paths. [NoFW]
- 4.8.12 Memory Consolidation Data Leakage Through Shared Context: Consolidation merges multiple independent reasoning traces into a single consensus output stored for retrieval. Consolidated memory containing summaries of multiple reasoning paths may inadvertently preserve sensitive information from original traces. In multi-agent systems sharing consolidated memory, all agents accessing that memory receive merged sensitive information from multiple sources, creating mixed-source sensitive data with broader leakage scope than any individual path access. [NoFW]
- 4.8.13 Intermediate Reasoning Trace Probabilistic Recall Vulnerability: Self-Consistency maintains multiple reasoning chains during generation. If these chains are partially preserved or included in logs, probabilistic recall mechanisms during multi-agent coordination may retrieve sensitive intermediate steps. In multi-agent systems, intermediate reasoning traces logged during Self-Consistency execution become queryable through downstream agents' context retrieval, enabling sensitive data leakage through probabilistic recall across agent boundaries. [NoFW]
- 4.8.14 Quality Score Metadata Leakage Via Preserved Paths: When Self-Consistency paths are preserved with quality scores, the metadata becomes information about which paths contained useful information. Over time, access patterns to high-quality preserved paths reveal domain-specific patterns, creating side-channel leakage of organizational reasoning patterns. In multi-agent systems sharing quality metadata, cross-agent access pattern analysis reveals organizational insights that single-agent quality metadata cannot expose. [NoFW]
- 4.8.15 Failure Case Logging Data Leakage Through Difficulty Classification: Failed Self-Consistency cases with low-confidence voting or no convergence are typically logged with full reasoning traces for debugging. If these logs are accessed by downstream agents for learning from failures, sensitive information from failed reasoning paths leaks. In multi-agent learning-from-failure systems, one agent's failed Self-Consistency reasoning becomes training data for other agents, leaking sensitive information through the failure documentation pipeline. [NoFW]
- 4.8.16 Consolidated Abstractions as Information Aggregation for Leakage: Consolidation summarizes raw episode details into abstractions stored in semantic memory that preserve enough information for reconstruction of underlying episodes. Attackers accessing abstraction records can infer underlying episode details. Multi-agent organizations with shared semantic knowledge built from consolidated abstractions create centralized repositories aggregating organization-wide information, enabling large-scale inference beyond what any single-agent system's local semantic memory would expose. [NoFW]
- 4.8.17 Graph Relationship Traversal for Operational Pattern Reconstruction: Graph databases storing episode relationships enable traversal that reconstructs operational sequences. Attackers traversing relationships can reverse-engineer operational workflows and decision patterns. While single agents' graph relationships reflect one agent's experience, multi-agent relationship graphs reflect entire organizational structure, enabling comprehensive operational intelligence reconstruction from the combined traversal of all agents' episodic relationships. [NoFW]
- 4.8.18 Deduplication Metadata as Statistical Information Leakage: Deduplication records which episodes are similar to each other, enabling frequency analysis that reveals which operations are routine versus rare. Attackers inferring deduplication patterns can determine threat model structures through statistical inference. Multi-agent aggregated deduplication statistics reveal organization-wide patterns rather than the per-agent patterns that single-agent deduplication statistics expose. [NoFW]
- 4.8.19 Temperature/Sampling Artifacts in Episodic Recall Creating Information Channels: Retrieval uses top-k from embedding similarity whose exact scores depend on temperature and noise during embedding generation. Attackers analyzing variance in repeated retrievals can infer underlying embedding structures and episode properties. Multi-agent systems with billions of retrievals create statistical patterns exploitable for inferring organization-wide patterns, a scale of inference not achievable against isolated single-agent embedding spaces. [NoFW]
- 4.8.20 RAG Relevance Scores as Probabilistic Information Leakage: Semantic similarity scores indicate how relevant knowledge base documents are to a query. Attackers can infer document content by analyzing relevance scores across multiple queries — probing with known-terminology queries to confirm presence and approximate characteristics of specific documents. In multi-agent systems where multiple agents share the same semantic index, each agent provides an independent observation channel enabling systematic corpus enumeration through a low-privilege information extraction path. [NoFW]
- 4.8.21 Semantic-to-Working Augmentation Information Leakage During Knowledge Distillation: When semantic knowledge is extracted from specific working memory episodes — generalizing "User_12345 solved problem via method_X" to "users typically solve via method_X" — distillation creates leakage of information from private episodes into shared semantic knowledge. In multi-agent systems, semantic memory becomes a privacy breach channel where information from episodic memory leaks through the distillation process, affecting all agents accessing shared semantic memory unlike singular semantic abstraction from single episodes. [NoFW]
- 4.8.22 Knowledge Graph Edge Weight Leakage via Embedding Inference: Knowledge graphs store weighted relationships enabling inference of sensitive associations. Embeddings trained on knowledge graphs may leak relationship weights through probabilistic generation. Multi-agent systems using shared knowledge graph embeddings create centralized leakage sources where attacks on embedding inference extract sensitive relationship data affecting the entire agent ensemble, unlike single-graph embeddings that leak structure only locally. [NoFW]
- 4.8.23 Few-Shot Demonstration Contamination with Leaked Sensitive Data: Few-shot demonstrations used for prompt learning may contain sensitive data from prior demonstrations, and attackers can craft prompts triggering probabilistic recall of sensitive demonstration data. Multi-agent shared demonstration pools create centralized repositories of accumulated sensitive data from all agents' prior demonstrations, enabling attackers accessing the pool to extract sensitive data at scale rather than from one agent's isolated demonstration set. [NoFW]
