# 12.3.34 Working Memory Content Monitoring Gaps [NoFW]

Working memory maintains intermediate facts during inference, but in multi-agent systems with shared working memory, tracking which agents retrieve and use which facts becomes opaque; monitoring cannot determine which agent accessed which fact and for what purpose in shared visibility contexts, and attackers exploit this opacity by poisoning working memory knowing that attribution of the poisoned fact to a specific agent becomes unclear in shared memory monitoring.
