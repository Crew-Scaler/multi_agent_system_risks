# 12.4.18 Trace Filtering as Selective History Manipulation [NoFW]

Trace filtering tools become attack vectors in multi-agent systems when attackers exploit filtering by poisoning span metadata. Attackers cause critical tool invocations to appear as non-error spans that filtering removes from oversight views, hiding evidence of unauthorized executions from downstream agents querying filtered traces. Unlike singular trace filtering affecting one agent's analysis, shared multi-agent trace repositories enable one metadata poisoning to affect all trace consumers simultaneously.
