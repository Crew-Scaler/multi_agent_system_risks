# 12.4.7 Prometheus Alerting Rule Threshold Gaming Creating Detection Blind Spots [NoFW]

Attackers exploit Prometheus alerting duration requirements by causing degradation that spikes for 4 minutes and recovers for 2 minutes, staying below the 5-minute sustained threshold requirement; in multi-agent fleets, rotating degradation across agent subsets ensures no single agent sustains degradation long enough to trigger per-agent alerts while maintaining continuous fleet-wide user impact.
