# 12.5.86 Cross-Agent Coordination Timing Blind Spots [NoFW]

Per-agent latency monitoring does not track inter-agent communication timing patterns; adversaries who exploit kernel timing or KV cache characteristics to create specific latency patterns in one agent that trigger malicious behavior when propagated to a dependent agent execute timing-based cross-agent attacks that are completely invisible to individual-agent telemetry.

## Supporting Evidence

- Linke Song et al.. 2025. The Early Bird Catches the Leak: Unveiling Timing Side Channels in LLM Serving Systems. *IEEE Transactions on Information Forensics and Security (TIFS)*. https://doi.org/10.1109/tifs.2025.3622954
  - [→ 2409.20002.md](../../papers/2409.20002.md)
  - "In our research, for the first time, we discovered a set of new timing side channels in LLM systems, arising from shared caches and GPU memory allocations, which can be exploited to infer both confidential system prompts and those issued by other users."
  - "However, memory sharing, cache contention and eviction, and GPU scheduling among different users and applications can interfere with user requests, creating observable timing side channels."
