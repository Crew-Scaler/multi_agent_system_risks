# 12.5.51 Prometheus Scrape Success Rate Hiding Agent-Level Failures [NoFW]

Prometheus tracks scrape success indicating that a metrics endpoint responded, but does not validate that metrics values are correct; agents that fail internally but maintain a functioning health check endpoint report successful scrapes while emitting incorrect metrics, and in multi-agent fleets this creates a monitoring blind spot where scrape success is incorrectly used as an agent health proxy.
