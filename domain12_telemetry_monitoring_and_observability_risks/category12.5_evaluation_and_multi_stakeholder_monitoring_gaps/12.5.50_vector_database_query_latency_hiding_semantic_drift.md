# 12.5.50 Vector Database Query Latency Hiding Semantic Drift [NoFW]

Vector database monitoring tracks query latency as a health proxy, but semantic drift from poisoned embeddings or corrupted HNSW graph structures causes fast queries that return wrong results; agents querying a semantically corrupted vector database experience correct latency metrics while operating on degraded retrieval content, and in multi-agent deployments all agents share this false health signal.
