# 12.2.71 Rule Modification Detection Gaps in Shared Repositories [NoFW]

Rule changes in shared repositories may not trigger consistent monitoring across all agents; when rules in shared repositories are modified, different agents may be at different stages of rule application creating race conditions, and monitoring that tracks individual agent rule states cannot detect that some agents apply old versions while others apply new versions, creating inconsistent policy enforcement exploitable by attackers.
