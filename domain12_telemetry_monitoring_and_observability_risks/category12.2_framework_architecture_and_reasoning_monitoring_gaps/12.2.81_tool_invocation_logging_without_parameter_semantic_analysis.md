# 12.2.81 Tool Invocation Logging Without Parameter Semantic Analysis [NoFW]

Tool call logs record function name and parameter values as opaque data rather than analyzing whether parameter values are semantically appropriate given the current conversation context, making parameter injection attacks invisible.

## Supporting Evidence

- Adam AlSayyad, Kelvin Yuxiang Huang, and Richik Pal. 2026. AgentTrace: A Structured Logging Framework for Agent System Observability. In *AAAI 2026 Workshop on Language Model Agent Systems (LaMAS)*. https://arxiv.org/abs/2602.10133
  - [→ 2602.10133.md](../../../papers/2602.10133.md)
  - "Crucially, contextual traces manage tool invocations and data access operations (reads and writes), linking agent reasoning with its external interactions."
- Juhee Kim et al.. 2026. The Attack and Defense Landscape of Agentic AI: A Comprehensive Survey. In *USENIX Security 2026*. https://arxiv.org/abs/2603.11088
  - [→ 2603.11088.md](../../../papers/2603.11088.md)
  - "They collect logs of agent actions (e.g., tool calls, file modifications) using services like OpenTelemetry [117] or PostHog [123], enabling post-hoc manual review, but lack detections for suspicious patterns or anomalies."
